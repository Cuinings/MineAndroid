## Fragment 切换策略三种实现
- 在 `demo/app-test` 模块实现了三套 Fragment 切换策略对比演示
- **共享 Fragment**：`HomeFragment`(点击计数+输入)、`MessageFragment`(消息计数)、`ProfileFragment`(创建时间戳)
- **StrategyA_ShowHideActivity**：add + show/hide，全驻留零重建
- **StrategyB_AttachDetachActivity**：add + attach/detach，实例保留 View 回收
- **StrategyC_ReplaceActivity**：replace，全量重建
- 通过 `MainActivity` 三个按钮入口，共用同一套 Fragment 和布局

## FocusScaleView 深度修复 — focusChangeReceived + isScaleActive 状态机
- **第一轮 (07-08)**: 用 `focusChangeReceived` 跳过首次 `onFocusChanged`，用 `isScaleActive` 替代逐帧 `scaleX` 浮点比较
- **第二轮 bug (07-09)**: 失焦动画 (`target=1.0f`) 结束后再获焦 → `animateToScale(focusScale)` 内 `scaleAnimator?.cancel()` 触发旧 animator 的 `onAnimationEnd` → `target==1.0f` 分支把刚设置的 `isScaleActive=true` 又重置为 false → 新放大动画期间 `drawChild` 不做反向补偿 → child 被放大而非保持原大小
- **修复**: `cancel()` 前先 `removeAllUpdateListeners()` + `removeAllListeners()`，防止旧 animator 的 `onAnimationEnd` 被连带触发误关 `isScaleActive`

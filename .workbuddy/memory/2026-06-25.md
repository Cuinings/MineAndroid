## BackgroundBlurDrawable 实现

基于 FrostedAnimatedGlowView 架构，实现了新的背景模糊方案：

### 新增文件
1. **BackgroundBlurDrawable.kt** (`core/core-ui/.../frosted/`)
   - 封装 Android 12+ 系统隐藏 `android.graphics.drawable.BackgroundBlurDrawable`
   - 一次性反射（companion object init），零 per-call 开销
   - `setBlurRegion(Rect)` 仅更新坐标，零分配、零 invalidate
   - 内部 Aggregator 在下一帧将区域转 float[] 下发 SurfaceFlinger → "0 延时"

2. **BackgroundBlurViewHelper.kt** (`core/core-ui/.../frosted/`)
   - 将 BackgroundBlurDrawable 附加到任意 View
   - 自动设置 Window blurBehindRadius + FLAG_BLUR_BEHIND
   - 内置 OnScrollChangedListener + OnGlobalLayoutListener 位置追踪
   - 快捷工厂方法 `BackgroundBlurViewHelper.attachTo(view, radius)`

### 修改文件
3. **StatefulGlowView.kt** — 新增 `protected fun setBackgroundInternal()`，允许子类绕过 setBackground 限制
4. **FrostedAnimatedGlowView.kt** — 新增 BackgroundBlurDrawable 模式：
   - `useBackgroundBlurMode` 属性（XML: `fagv_useBackgroundBlurMode`）
   - `backgroundBlurRadius` (0-25), `backgroundBlurCornerRadius`
   - 自动管理 Window 模糊管道 + drawable 生命周期
   - 位置追踪绑定到现有 onLayout / setTranslationX/Y / onSizeChanged
   - dispatchDraw 中跳过 capture+blur 管线
5. **attrs.xml** — 新增 3 个 fagv_ 前缀属性

### 架构对比
| 方案 | 延时 | CPU | 内存 | 适用 |
|------|------|-----|------|------|
| FrostedAnimatedGlowView (capture+blur) | 1-2帧 | 截屏+模糊 | Bitmap | 通用 |
| WallpaperBlurView (Compositor) | 0帧 | 0 | 0 | 仅静态壁纸 |
| **BackgroundBlurDrawable** | **0帧** | **0** | **0** | **任意背景** |

### 核心机制
- View 移动 → `updateBackgroundBlurRegion()` → `blurDrawable.setBlurRegion(l,t,r,b)`
- 仅 Rect 坐标赋值，不 new、不 setBackground、不 invalidate
- 系统 Aggregator 帧级批处理 → SurfaceFlinger 直接合成 → 零延时

## BackgroundBlurDrawable 自实现 (16:04)

### 问题
系统 `android.graphics.drawable.BackgroundBlurDrawable` 在部分 OEM ROM 上不存在（小米/OPPO 等移除），反射创建失败 → 模糊不显示。

### 解决方案
将 BackgroundBlurDrawable 从反射工具类改为**自定义 Drawable 实现**：
- 全屏截屏（SurfaceControl.screenshot → PixelCopy 兜底）
- RenderNode + `RenderEffect.createBlurEffect()` → GPU 模糊
- `setBlurRegion(l,t,r,b)` → 仅 Rect 坐标赋值，零分配
- `draw()` → canvas.drawRenderNode(blurNode, cropRect) → 裁剪渲染
- 每 2s Handler.postDelayed 刷新全屏截屏

### 关键改动
- FrostedAnimatedGlowView: 移除 Window blurBehindRadius/FLAG_BLUR_BEHIND、移除 cross-window 检测、移除反射逻辑
- BackgroundBlurViewHelper: 同步 API 更新

### 零延时机制
- View 移动 → setBlurRegion(l,t,r,b) → Rect 赋值（O(1)）
- 下一帧 draw() 从已模糊的全屏 capture 中 GPU 裁剪 → 近乎零开销
- 不重截屏、不重模糊、不新建对象

## 修复反射路径 (16:27)

### 问题
反射查找 `android.graphics.drawable.BackgroundBlurDrawable` 失败，
实际路径是 **`com.android.internal.graphics.drawable.BackgroundBlurDrawable`**。

### 修复
- 优先尝试 `com.android.internal.graphics.drawable.BackgroundBlurDrawable`
- 兜底 `android.graphics.drawable.BackgroundBlurDrawable`
- 两者都失败 → FallbackBlurDrawable（全屏截屏+RenderEffect 自定义实现）
- 截屏改为异步（handler.post），避免 PixelCopy 主线程死锁
- PixelCopy 改为 AtomicReference + CountDownLatch 模式，确保跨线程安全

## HomeActivity WallpaperBlurView 拖拽修复 (17:46)

### 问题
`setupDragSupport()` 中 `ACTION_DOWN` 返回 `false`，导致 Android 触摸系统判定该 View 不处理手势，
不再分发后续 `ACTION_MOVE`/`ACTION_UP` 事件 → 拖拽逻辑永远不触发。

### 修复
- `ACTION_DOWN` 改为返回 `true`，手动 `view.isPressed = true`
- `ACTION_UP`/`ACTION_CANCEL` 统一返回 `true`，始终 `view.isPressed = false`

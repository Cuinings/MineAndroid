# 2026-06-03 工作日志

## WallpaperBlurView 动态壁纸三 bug 修复
- **现象1**：动态壁纸模糊内容显示静态壁纸（内容错误）
- **现象2**：修改后整个 Activity 全局模糊
- **现象3**：显示器上显示黑屏

- **根因分析（3个关键 bug）**：
  1. **FLAG_BLUR_BEHIND 顺序错误**：`activateCompositorBlur()` 中先设了 `FLAG_BLUR_BEHIND` + `blurBehindRadius`，再检测 `isLiveWallpaper()`。检测到动态壁纸后调 `activateWallpaperBitmapBlur()`，但 `FLAG_BLUR_BEHIND` 仍留在窗口上 → SurfaceFlinger 模糊整个壁纸层 → 全局模糊
  2. **LAYER_TYPE_HARDWARE 导致截屏黑屏**：`scheduleScreenCapture()` 中 `isCapturingScreen=true` 让 dispatchDraw 跳过绘制，但 View 有 `LAYER_TYPE_HARDWARE`（硬件层是不透明 buffer），即使不画任何内容硬件层本身也不透明 → SurfaceControl.screenshot() 捕获到黑色 View 区域而非壁纸
  3. **截屏失败无限循环**：SurfaceControl.screenshot() 反射失败 → screenCaptureBitmap 始终 null → updateWallpaperBlurNode() 反复调 scheduleScreenCapture() → 无限循环

- **修复**：
  1. **isLiveWallpaper() 检查移到 FLAG_BLUR_BEHIND 之前**：动态壁纸直接走 bitmap 模糊路径，不设 FLAG_BLUR_BEHIND
  2. **截屏时临时 LAYER_TYPE_NONE**：setLayerType(NONE) → View 真正透明 → 截屏能捕获壁纸 → 截完恢复 HARDWARE
  3. **截屏健壮性**：
     - 尝试 3 种 SurfaceControl.screenshot() 签名（6参/5参/2参兼容）
     - 尝试 2 种 getDisplayToken 方法（getInternalDisplayToken/getPhysicalDisplayToken）
     - 500ms 最小截屏间隔节流
     - 连续 5 次失败后降级到 WallpaperManager.getDrawable()（静态壁纸缓存）
     - 等待 2 帧 Choreographer 确保 LAYER_TYPE_NONE 已生效
  4. **动态壁纸截图直接全图绘制**：screenCaptureBitmap 已是 View 区域截图，不需要再做 screenX/screenY 缩放裁剪
- **编译验证**：BUILD SUCCESSFUL

## 动态壁纸 Bitmap 模糊源修复：统一使用 getDrawable()

- **现象**：修改后动态壁纸完全无模糊（isLiveWallpaper 检查前置修复 + 截屏改进后，截屏失败导致无 bitmap 源）
- **根因**：`SurfaceControl.screenshot()` 反射不兼容用户设备 ROM → `screenCaptureBitmap` 始终 null → `updateWallpaperBlurNode()` 只等截屏结果 → RenderNode 从未创建
- **修复**：`updateWallpaperBlurNode()` 统一使用 `WallpaperManager.getDrawable()` 作为 bitmap 模糊源：
  - 静态壁纸：`getDrawable()` 返回当前壁纸 ✅
  - 动态壁纸：`getDrawable()` 返回系统底层静态壁纸缓存，25px 高斯模糊后视觉效果可接受
  - 只有 `getDrawable()` 返回 null 时才尝试 screenCapture 后备
  - 裁剪逻辑统一：始终使用 screenX/screenY 缩放映射
- **同时简化**：`onPositionChanged()` 动态壁纸路径不再特殊处理，统一 `wallpaperBlurDirty=true + invalidate()`
- **截屏基础设施保留**（`scheduleScreenCapture`、`captureScreenRegion`、`fallbackToStaticWallpaperCapture`）作为可选增强路径
- **编译验证**：BUILD SUCCESSFUL

## 动态壁纸模糊源修正：截屏为主+getDrawable垫底双路径

- **用户反馈**：getDrawable() 返回旧静态壁纸，不是动态壁纸实时内容，必须模糊实时动态壁纸
- **修复**（3处改动）：
  1. **`updateWallpaperBlurNode()` 双路径**：动态壁纸时优先用截屏，截屏未完成时 getDrawable 垫底（不 return），异步截屏
  2. **`scheduleScreenCapture()` 截屏回调直接替换 wallpaperBitmapRaw**：截屏成功后 wallpaperBitmapRaw = captured
  3. **`captureScreenRegion()` 容错**：token 获取失败不致命 → 仍试无 token 方法；新增 `getDisplayToken(int)` 方法
- **核心思路**：垫底立即可见 + 截屏完成后无缝替换
- **编译验证**：BUILD SUCCESSFUL

## V3 架构重设计：静态 Composer Blur + 动态全屏截屏裁剪

**新架构：**

| 壁纸类型 | 模糊路径 | 延时 | 区域 |
|---------|---------|------|-----|
| 静态 | Compositor Blur + WallpaperMaskDrawable | 0 | ✅ View rect |
| 动态 | SurfaceControl.screenshot() 全屏截 → 裁剪 → RenderEffect | 0（移动）2s（刷新） | ✅ View rect |

**核心实现：**
- `activateLiveWallpaperCapture()`: OnPreDrawListener 首帧前截全屏（View 透明时捕获 → 无闪烁）
- `captureFullScreen()`: 系统 jar 直接调 SurfaceControl.screenshot() + 反射 fallback
- 移动零延时: fullScreenCapture 尺寸=屏幕像素→scaleX/Y=1→裁剪=View屏幕坐标，仅 dirty+invalidate
- 定时刷新: `scheduleFullScreenRefresh()` 每 2s 用 OnPreDrawListener 机制
- 移除 getDrawable() 动态壁纸备选逻辑
- **编译验证**：BUILD SUCCESSFUL

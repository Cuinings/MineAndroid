## 修复 currentState 概率性误置 DISABLED
- **根因**：`isEnabled` 递归检查祖先链，父级临时禁用（布局/RecyclerView 回收等场景）时 `!isEnabled=true`，触发 `drawableStateChanged`/`setPressed`/`onFocusChanged` 中误判，将状态错误翻为 DISABLED
- **修复**：新增 `selfEnabled` 布尔字段跟踪本 View 自身的 enable 状态（不受父级影响）
- 4 处 `!isEnabled` → `!selfEnabled`：`drawableStateChanged()` / `setPressed()` / `onFocusChanged()` / 状态判断逻辑
- `setEnabled(enabled)` 中在 `super.setEnabled()` 之前更新 `selfEnabled`，避免 `drawableStateChanged()` 回调读到旧值

## 状态机乒乓切换修复
- **根因**：`setPressed/setEnabled/onFocusChanged/setSelected` 各自独立 `canCompute(新态)→animateStateChange()`，然后 super 调用再触发 `drawableStateChanged()` 二次判态，产生冗余/冲突动画：
  - #1 `setPressed(false)` 且 `hasFocus()`：PRESSED → NORMAL → FOCUSED（两次动画，闪烁）
  - #2 `setEnabled(true)` 且 `hasFocus()`：DISABLED → NORMAL → FOCUSED（三连跳）
  - #3 `setSelected(false)` 且 `hasFocus()`：SELECTED → FOCUSED → NORMAL（最终停在 NORMAL 而非 FOCUSED，语义错误）
- **修复**：`drawableStateChanged()` = 状态机唯一入口；4 个 override 只调 super，不再独立触发动效
  - `setPressed/setSelected/onFocusChanged` → 仅 `super.xxx()` 
  - `setEnabled` → `selfEnabled = enabled; super.setEnabled(enabled)`

## normal 状态自动级联 fallback
- **需求**：只设 normal 状态值时，其他状态自动继承，避免逐状态重复配置
- **实现**：三数据类各加 `fallbackFromNormal()`：
  - `StateWidth`: 非 normal 状态值为 `0f` → 复制 normal
  - `StateColors`: 非 normal 状态值为 `TRANSPARENT` → 复制 normal
  - `StateDrawables`: 非 normal 状态值为 `null` → 复制 normal
- **触发**：init 中每类属性用 `hasValue(sgv_xxxNormal)` 判是否显式设置，设置过才级联
- **懒初始化保护**：lazy 设的默认值（如 `strokeColors.focused=#7FFFFFFF`）不是 TRANSPARENT，不会被 fallback 覆盖

## Activity 切换状态残留修复
- **根因**：Activity 切换时 `onDetachedFromWindow` 取消 animator，但 paint 值停在插值中途（如 NORMAL→FOCUSED 50% 处）。重 attach 后 `drawableStateChanged` 判定同状态（`currentState==targetState`），guard skip 不触发新动画，视觉残留在中间值
- **修复**：`onDetachedFromWindow` 末尾追加 `applyStateInstantly(currentState)`，将 paint snap 到 currentState 终点值，确保下次 attach 时视觉正确

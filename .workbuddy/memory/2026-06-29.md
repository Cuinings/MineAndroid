# 2026-06-29 工作日志

## BlurView 新增 sourceBitmap 外部 Bitmap 模糊路径

### 新增属性
- `sourceBitmap: Bitmap?` — 外部传入源 Bitmap，非空且 blurEnabled=true 时优先使用，跳过 SYSTEM/FALLBACK
- `blurScaleFactor: Float = 0.1f` — 缩放因子，Bitmap 和 View 坐标等比缩放后裁剪

### 新增 BlurMode.BITMAP
- 优先级最高：activateBlur() 中 sourceBitmap 非空即走 BITMAP 路径
- 设为 null 自动恢复 SYSTEM/FALLBACK 路径

### SourceBitmapBlurDrawable 内部类
- `attach()`: 缩放 Bitmap → 计算裁剪区域 → 构建 RenderNode（RenderEffect 模糊）
- `draw()`: 直接 drawRenderNode（RenderNode 已裁剪对齐 View 尺寸，无需 translate）
- `updateRegion()`: View 移动时更新 cropRect + 重建 RenderNode
- `setBlurRadius()`: 仅更新 RenderEffect，零重录
- `invalidateCache()`: scale 变更后重建缩略图 + RenderNode
- 位置变化：OnScrollChangedListener + OnGlobalLayoutListener 追踪

### SourceBitmapBlurDrawable 修复
- `draw()`: 删除多余 `canvas.translate(-screenX, -screenY)`（RenderNode 已裁剪对齐）
- `buildScaledBitmap()`: 始终创建缩放副本，不再直接引用外部 bitmap（防止 detach 时误 recycle）
- `draw()`: 增加 `source.isRecycled` 保护
- 删除无用 `regionDirty` 字段、未使用的 import（RectF, createBitmap, scale）

## BlurView 新增 blurredDrawable 属性
- `blurredDrawable: Drawable?` — 外部传入预渲染 Drawable，非空时直接设为 View 背景，完全跳过所有模糊路径
- XML: `app:bv_blurredDrawable="@drawable/xxx"`
- `applyBlurredDrawable()`: 设置时 deactivate+LAYER_TYPE_NONE+setBackgroundInternal
- 设为 null 时恢复 blurEnabled 标记的正常模糊流程
- `activateBlur()` / `onAttachedToWindow()` 入口增加 blurredDrawable 优先判断
- attrs.xml 新增 `bv_blurredDrawable` (reference 格式)

## BlurView 新增 blurredWallpaperBitmap 属性
- `blurredWallpaperBitmap: Bitmap?` — 外部预模糊壁纸 Bitmap，非空时转 BitmapDrawable 设为背景
- `applyWallpaperBitmap()`: 转换并 setBackgroundInternal，LAYER_TYPE_NONE
- 优先级: blurredDrawable > blurredWallpaperBitmap > sourceBitmap > SYSTEM > FALLBACK

## BlurPopupWindow + PopupWindowBlurHelper 实现（基于 BasicPopupWindow）

### 架构
- `BlurPopupWindow` 继承 `BasicPopupWindow`，零侵入
- `PopupWindowBlurHelper` 单例引擎，可独立用于任意 PopupWindow
- 扩展函数 `PopupWindow.enableBlur()` / `disableBlur()`

### 模糊路径
1. **主路径 Window 级 Cross-Window Blur（API 31+，零延时）**
   - `showAtLocation()` 后获取 PopupDecorView → LayoutParams
   - `blurBehindRadius` + `FLAG_BLUR_BEHIND`（公开 API，无需 system.jar）
   - `WindowManager.isCrossWindowBlurEnabled` 检测 OEM 支持
   - SurfaceFlinger GPU 原生模糊 → 零延时、零绘制

2. **Fallback 全屏截图 + RenderEffect（OEM 禁用时）**
   - SurfaceControl.screenshot() 3 种反射签名 + PixelCopy 多策略截图
   - RenderEffect.createBlurEffect() GPU 模糊
   - 2s 定时刷新应对下方内容变化
   - 预留 system.jar 直接调用入口（`captureScreenDirect()`，注释即启用）

### 关键属性
- `blurRadius: Int [0, 25]` — 模糊强度
- `dimAmount: Float [0, 1]` — 遮罩深度
- `blurEnabled: Boolean` — 开关
- 运行时修改自动生效（reapplyBlur）

### 生命周期
- `showAtLocation()` → `applyBlur()` → `PopupWindowBlurHelper.configure()`
- `dismiss()` → `removeBlur()` → 清理 Window blur 属性 + Fallback Drawable
- blurApplied 标志位防重复

### Application Context 修复（2026-06-29）
- **问题**：用 Application Context 创建时，`FallbackBlurDrawable.captureScreen()` 中 `pw.contentView?.context` 拿到的也是 Application Context → `resolveActivity()` 返回 null → PixelCopy 截图策略失效 → fallback 无模糊
- **修复**：新增公共函数 `Context.toActivity()` 解析 Activity；`BlurPopupWindow` 构造器自动调用存为 `resolvedActivity`；`configure()` 向下透传给 `FallbackBlurDrawable(activity=...)` 直接使用
- **排查**：查看 `adb logcat -s PwBlurHelper` 可确认走的主路径还是 fallback，以及 fallback 三策略（SurfaceControl v1/v2/v3 + PixelCopy）哪个成功

### PopupDecorView 遍历修复（2026-06-29）
- **问题**：`pw.contentView?.parent` 返回 PopupBackgroundView（背景 wrapper），不是 PopupDecorView
  - PopupWindow 有背景时实际结构: PopupDecorView → PopupBackgroundView → contentView
  - PopupBackgroundView 的 layoutParams 是 FrameLayout.LayoutParams，不是 WindowManager.LayoutParams → `applyWindowBlurInternal` 中 `lp as? WindowManager.LayoutParams` 返回 null → 主路径默默失败
- **修复**：新增 `getPopupDecorView()` 向上遍历最多 3 层，查找持有 WindowManager.LayoutParams 的 View
  - `applyWindowBlurInternal` / `clearWindowBlur` / `FallbackBlurDrawable` 全部改用此方法

### Fallback OnPreDrawListener 截图修复（2026-06-29）
- **问题**：`attach()` 中 `handler.post { doCapture() }` 在弹框已显示后异步执行 → 截图包含弹框自身 → 递归模糊
- **修复**：改用 `ViewTreeObserver.OnPreDrawListener` 在 PopupDecorView 首帧绘制前同步触发截图
  - OnPreDrawListener 只执行一次，执行后自动移除
  - 首帧 screen 尚不含弹框 → 正确模糊背景
- **SurfaceControl 反射日志增强**：每个 variant 失败时记录异常类名（`NoSuchMethodException` / `InvocationTargetException` 等），方便诊断 hidden API 限制

### BlurPopupWindow 架构重写 — PixelCopy 异步捕捉（2026-06-29 第三次）
- **核心变更**：放弃对 PopupWindow sub-window 的 FLAG_BLUR_BEHIND 依赖，改用异步 PixelCopy 截图 → RenderEffect 模糊 → BitmapDrawable 背景
- **流程**：`showAtLocation()` → 先试 Window 级模糊（零延时） → 失败则异步 PixelCopy Activity Window → 独立线程 RenderNode 模糊 → 回调主线程 setBackgroundDrawable
- **中位图降采样**：0.25x 缩放 → 加速 RenderEffect 模糊 → 放大回原始尺寸（GPU 双线性过滤天然平滑）
- **PopupWindowBlurHelper 简化**：移除 FallbackBlurDrawable / SurfaceControl 反射 / OnPreDrawListener，保留 `applyWindowBlur()` + 新增 `applyPixelCopyBlur()`

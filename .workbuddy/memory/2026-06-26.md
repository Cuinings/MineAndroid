## FrostedAnimatedGlowView 重构 — 纯 bitmap 模糊

### 根因
FrostedAnimatedGlowView 和 WallpaperBlurView 各自定义了 `WallpaperMaskDrawable` + `MaskEntry`，通过 `activityMasks` 静态 Map 管理 decorView 背景。两个类同时激活时，先激活的 mask 占据 decorView，后激活的增加 refCount 但不替换 mask。导致 WallpaperBlurView 的 `updateMaskHole()` 实际操作的是 FrostedAnimatedGlowView 的 mask，但坐标是 FrostedAnimatedGlowView 的位置 → holes=0。

### 修复
删除 FrostedAnimatedGlowView 中整个 mask 系统（~200 行），仅保留 bitmap 模糊路径：

```
activateBlur: WallpaperManager.getDrawable().toBitmap()
  → RenderNode 全屏录制 + RenderEffect.createBlurEffect()

dispatchDraw:
  canvas.translate(-screenX, -screenY)  ← 零延时裁剪
  canvas.drawRenderNode(mBlurNode)      ← 已有模糊内容
```

### WallpaperBlurView 精简
删除了所有装饰效果（overlay/glow/stroke/topHighlight/state动画），仅保留：
- blurEnabled / blurRadius / WallpaperMode
- Compositor Blur 主路径（blurBehindRadius + Mask）
- Bitmap + RenderEffect 降级路径
- 动态壁纸全屏截屏路径
- 位置追踪 + 壁纸监听

代码从 ~2110 行精简到 ~450 行。

### 文件变更
- FrostedAnimatedGlowView.kt: 重写，纯 bitmap 模糊 + 父类装饰效果
- WallpaperBlurView.kt: 删除装饰效果，仅保留模糊
- StatefulGlowView.kt: 保留 setBackgroundInternal() 方法

## 新增 BlurView — 0延时内容模糊视图

### 设计
新建 `BlurView` 继承 `AnimatedStatefulGlowView`，使用 `BackgroundBlurDrawable` 反射系统实现内容模糊：
- **主路径 (SYSTEM)**: `Window.setBackgroundBlurRadius()` + `ViewRootImpl.Aggregator` → `createBackgroundBlurDrawable()` → SurfaceFlinger GPU 模糊 → Aggregator 自动追踪位置 → 0延时
- **降级 (FALLBACK)**: `FallbackBlurDrawable` → `SurfaceControl.screenshot()` 全屏截取 + `RenderEffect.createBlurEffect()` → canvas.translate 裁剪 → 近0延时

### 关键设计
- 使用 `setBackgroundInternal()` 绕过 StatefulGlowView 的 setBackground 限制
- System 路径无需位置追踪（Aggregator 自动管理），Fallback 路径通过 OnScrollChangedListener + OnGlobalLayoutListener 更新 blur region
- 生命周期：onAttachedToWindow 激活 → onDetachedFromWindow 停用
- 自定义属性：`bv_blurEnabled` / `bv_blurRadius` / `bv_blurCornerRadius`

### 文件变更
- `BlurView.kt`: 新建（323行）
- `attrs.xml`: 新增 BlurView styleable（bv_ 前缀属性）

## BlurView SYSTEM 模式不显示模糊 — 修复

### 根因分析
1. **LAYER_TYPE_HARDWARE 阻断模糊（致命）**: StatefulGlowView.init() 设置 LAYER_TYPE_HARDWARE，View 渲染到离屏纹理，遮挡 SurfaceFlinger 模糊合成
2. **缺少 isCrossWindowBlurEnabled 检测**: OEM ROM 禁用跨窗口模糊时静默失效
3. **dispatchDraw 重复绘制**: BlurView.dispatchDraw 重复绘制 StatefulGlowView 已处理的装饰层

### 修复内容
- `activateBlur()`: 添加 `WindowManager.isCrossWindowBlurEnabled` 检测，OEM 禁用时直接降级 Fallback
- `onSystemBlurReady()`: 设 `setLayerType(LAYER_TYPE_NONE, null)` 移除硬件层
- `deactivateBlur()`: SYSTEM 退出时恢复 `LAYER_TYPE_HARDWARE`
- 移除 BlurView.dispatchDraw 覆写（父类链已处理全部装饰绘制）
- 清理不再需要的 import（Canvas, withClip）

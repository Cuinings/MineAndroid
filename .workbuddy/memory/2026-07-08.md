## FrostedVideoOverlay → BlurView.targetTextureView (TextureView 模糊完整历程)
- **路径 C 起点**: 新建 `FrostedVideoOverlay`，BackgroundBlurDrawable Aggregator per-View 模糊 — 显示壁纸非视频（TextureView 独立 Surface 不在 View 缓冲内）
- **路径 B 尝试**: BlurView 内 SurfaceControl 子层 `setRelativeLayer + setBackgroundBlurRadius` — 同样 Window 级模糊壁纸
- **根因确认**: `SurfaceControl.setBackgroundBlurRadius()` 无论作用哪个子层，SurfaceFlinger 最终都是 Window 后方内容模糊
- **最终方案**: BlurView 内 `TextureCaptureBlurDrawable` — 帧捕获 + RenderEffect GPU 模糊
  - `targetTextureView` 绑定 TextureView
  - 后台线程 `getBitmap()` → 降采样 0.25x → 裁剪 → `RenderEffect.createBlurEffect` → RenderNode 绘制
  - ~20fps 定时刷新，visibility 检查 (隐藏时 fall through 到 SYSTEM/FALLBACK)
  - BlurView.kt 新增 ~150 行；FrostedVideoOverlay.kt/demo 保留
- **影响**: Tegra/PowerVR/部分低端 Mali 可能因此黑屏，修复后消除风险

## GLTextureBlurOverlay 7 实例性能分析
- 致命瓶颈：7 独立 EGL 上下文 → 每帧 3.5-14ms 上下文切换开销
- 严重瓶颈：shader 动态分支 `if-continue` 在 GPU warp 锁步中失效，radius=8 实际采样 65 texel 而非期望的 17
- 纹理采样带宽：7 实例重叠区域 245 次/像素，60fps 时 ~13.5 GB/s 接近 LPDDR4x 极限
- 推荐优化：单一 GLSurfaceView + 多区域渲染（1 上下文、3 pass、shader 复用）

## CodecLoopVideoView 三个 Bug 修复
- **P0 循环 PTS 加速**: inputEOS 分支 loop 时未重置 `isFirstFrame=true`，导致第二圈 `basePtsUs` 沿用上一圈末尾大值，帧间隔计算为负、全部零延时渲染 → 视频极速播放
- **P0 线程安全**: `stopCodec()` 先 `codec.stop()` 后 `join()`，解码线程可能在 dequeue 时 codec 已 Stopped → `IllegalStateException`。修复为: isDecoding=false → interrupt → join(500) → stop
- **P1 Surface 缓冲**: 添加 `surfaceTexture?.setDefaultBufferSize(videoWidth, videoHeight)` 防止分辨率不匹配拉伸
## SystemBlurOverlay v2 — SurfaceControl 子层 + Buffer 方案
- **v1 失败原因**: `BackgroundBlurDrawable` 是 WINDOW 级跨窗口模糊，无法模糊同窗口内的 TextureView
- **v2 方案**: `SurfaceControl.Builder` 创建透明格式子层(z=1) + `Surface(SurfaceControl)` 提供 buffer + `setBackgroundBlurRadius`
- **关键**: 必须有 buffer 才能让 SurfaceFlinger 合成模糊；用 Hidden API `new Surface(SurfaceControl)` 给子层分配 buffer queue

## GLTextureBlurOverlay attachToGLContext 崩溃修复
- **崩溃**: `RuntimeException: Error during attachToGLContext`
- **根因**: TextureView 内部已将 SurfaceTexture attach 到自身 GL 上下文，GLTextureBlurOverlay 直接 attachToGLContext 时因 SurfaceTexture 只能绑定一个上下文而失败
- **修复**: 重构为自建 SurfaceTexture 方案 — GL 线程创建 `SurfaceTexture(texId)` → 暴露 `videoSurface` 供外部视频源渲染 → `attachToGLContext` 在自己上下文中正常执行
- **额外修复**: `createExternalTexture()` 和 `onDrawFrame` Pass1 的纹理目标从 `GL_TEXTURE_2D` 改为 `GL_TEXTURE_EXTERNAL_OES`（Mali 驱动 texture target mismatch 错误）
- **API 变更**: 删除 `bindTextureView()`，新增 `videoSurface: Surface?`；`CodecLoopVideoView` 新增 `outputSurface: Surface?` 属性

## FocusScaleView 首次加载 child 变小修复
- **Bug**: 首次加载时 Android 初始焦点分配触发 `onFocusChanged(true)` → `animateToScale(focusScale)` 设 scaleX>1.0 → `drawChild` 反向缩放与 RenderNode 正向缩放在首帧渲染管线中未完美抵消 → child 视觉上变小
- **修复思路**: "仅焦点变化时"才缩放计算 childView —— 初始焦点不算"变化"
- **修改** `core/core-ui/.../frosted/FocusScaleView.kt`:
  - 新增 `focusChangeReceived: Boolean`：首次 `onFocusChanged` 仅标记，跳过动画
  - 新增 `isScaleActive: Boolean`：`drawChild` 用布尔判断替代逐帧 `scaleX` 浮点比较，零开销
  - `animateToScale`: width<=0 时 `post {}` 延迟（安全网）+ `onAnimationEnd` 中 target==1.0 时关 `isScaleActive`
  - `onDetachedFromWindow` 重置所有状态（防 RecyclerView 复用残留）

## BlurView 动态壁纸黑屏修复（2026-07-01 下午）

### 问题
开启 BlurView 模糊后设置动态壁纸，模糊区域显示黑色而非模糊壁纸。

### 根因分析
| 视图 | 动态壁纸检测 | 处理方式 | 结果 |
|------|-------------|---------|------|
| WallpaperBlurView | 有 | WallpaperManager.drawable → bitmap 模糊 | 正常 |
| FrostedAnimatedGlowView | 有 | WallpaperManager.drawable → bitmap 模糊 | 正常 |
| **BlurView** | **无** | SYSTEM 失败 → FALLBACK (SurfaceControl.screenshot) | **黑色** |

SurfaceControl.screenshot() 只捕获应用层 Surface 缓冲区，动态壁纸（WallpaperService）渲染在独立 Surface 上，截图拿不到壁纸内容。

### 修复
在 `BlurView.activateBlur()` 中 BITMAP 路径后、SYSTEM 路径前插入 LIVE WALLPAPER 路径：
1. `BlurMaskManager.isLiveWallpaper(context)` 检测
2. `WallpaperManager.drawable` → `toBitmap()` 全屏分辨率获取壁纸帧
3. 复用已有 `SourceBitmapBlurDrawable` (BITMAP 管线) + `RenderEffect` GPU 模糊
4. `liveWallpaperRefreshRunnable` 每 2s 定时刷新（`Bitmap.sameAs()` 去重，无变化不重绘）
5. `deactivateBlur()` / `onDetachedFromWindow()` 中完整清理：停止刷新 + recycle bitmap

### 编译
- `:core:core-ui:compileDebugKotlin` 通过，无新增错误/警告

## BlurView sourceBitmap 黑屏修复（2026-07-01 下午）

### 问题
`sourceBitmap` 传入后 BITMAP 路径背景显示黑色。

### 根因
`SourceBitmapBlurDrawable.draw()` 中 `source.isRecycled` 检查错误：
- RenderNode 录制的是 `scaledBmp`（缩放副本），不是 `source`
- 外部回收 `source` 不影响已录制的 display list
- `source.isRecycled` → true 时 draw() 直接 return → 黑色

### 修复
1. `draw()`：移除 `source.isRecycled` 检查；改为检查 `scaledBmp` 状态，scaledBmp 无效时自动重建
2. `draw()`：增加 `cropRect.isEmpty` 防御检查；`hasDisplayList()=false` 时尝试立即 rebuild
3. `updateRegion()`：`scaledBmp` 为 null/回收时自动调用 `buildScaledBitmap()` 重建

### 编译
- compileDebugKotlin 通过

## BlurView sourceBitmap 诊断日志（2026-07-01 下午）

### 变更
为 `SourceBitmapBlurDrawable` (BITMAP 路径) 添加完整诊断日志链路，`DEBUG=true` 时覆盖：
- `activateBlur()`: BITMAP 入口参数 (src尺寸/scale/radius/view尺寸)、attach后状态
- `attach()`: 进入/完成状态
- `buildScaledBitmap()`: 缩放前后尺寸
- `updateRegion()`: 屏幕位置、裁剪计算、scaledBmp状态
- `buildBlurNode()`: RenderNode创建、RenderEffect、录制完成、hasDisplayList
- `draw()`: 每次跳过原因 (!attached/scaledBmp无效/node=null/cropRect空/noDL) 或成功绘制

日志 tag: `BlurView`，过滤 `SBD` 即可只看 BITMAP 管线日志

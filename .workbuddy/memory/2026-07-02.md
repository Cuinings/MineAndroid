## StatefulGlowView dispatchDraw 重构 — 修复 normal 状态不显示 bug

### 根因
`AnimatedStatefulGlowView.dispatchDraw` 调用 `super.dispatchDraw(canvas)` (StatefulGlowView) 先绘制装饰+children，然后自己再次绘制装饰层。导致：
1. **装饰层双绘**（overlay/glow/stroke/highlight 画两次）
2. **children 被第二次 overlay 覆盖**（如果 normal 状态有非透明 overlay，children 不可见）
3. **w/h<=0 时 `super.dispatchDraw` 被调用两次**
4. **状态动画 from 值竞态**：`animFromGlowRadius` 取 `glowRadii.get(currentState)`，但 `currentState` 已被前一次 `animateStateChange` 提前更新，不反映当前渲染值——导致快速连切状态时 glow 闪烁跳变

### 修复
1. **StatefulGlowView**: 提取 `drawAllDecorations(canvas, w, h)` 和 `dispatchDrawChildren(canvas)` 独立方法
2. **AnimatedStatefulGlowView**: dispatchDraw 改为 装饰→流光/刷新→dispatchDrawChildren，不再走父类 dispatchDraw
3. **FrostedAnimatedGlowView** + **FrostedAnimatedGlowView1**: 同上，改为 模糊背景→drawAllDecorations→流光/刷新→dispatchDrawChildren
4. **动画 from 值**: `animFromGlowRadius` 改用 `getCurrentGlowRadius()`（取动画器实时值），不再用 `glowRadii.get(currentState)`

### 文件
- `StatefulGlowView.kt`: +`drawAllDecorations` +`dispatchDrawChildren`；`animateStateChange` 修正 glow from 值
- `AnimatedStatefulGlowView.kt`: dispatchDraw 重构
- `FrostedAnimatedGlowView.kt`: dispatchDraw 重构
- `FrostedAnimatedGlowView1.kt`: dispatchDraw 重构

## StatefulGlowView stroke 消失修复 (v2, layer 拆分)

### 根因
v1 重构将全部装饰(含 stroke)放在 children 下方，而旧版 AnimatedStatefulGlowView 通过 double-draw 副作用将 stroke 置于 children 上方。children 有不透明内容时会盖住 stroke。

### 修复 — 分层绘制
三个子类 dispatchDraw 改为 4 层结构：
- **Layer 1** (底): 模糊背景 (仅 Frosted)
- **Layer 2**: overlay + innerGlow — children 下方作为背景
- **Layer 3**: children — 内容层
- **Layer 4** (顶): stroke + topHighlight + flow + refresh — children 上方确保描边始终可见

- `FrostedAnimatedGlowView.kt`
- `FrostedAnimatedGlowView1.kt`

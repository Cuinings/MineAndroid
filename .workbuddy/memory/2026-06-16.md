## CommonNode 聚合数据模型实现

### 背景
多类型独立 CRUD 通知（类型1/2/3...各自包含部门信息和用户信息），需聚合成统一的 `CommonNode`。

### 新增文件（`board/module/database/src/main/java/com/cn/board/database/`）

1. **CommonNode.kt** — 聚合实体
   - `NodeType` 枚举: GROUP（部门）/ MEMBER（用户）
   - `ResourceType` 枚举: TYPE_1(1), TYPE_2(2), TYPE_3(3)... 可扩展
   - `CommonNode` @Entity: 复合主键 `"{resourceType}_DEPT_{deptId}"` 或 `"{resourceType}_USER_{userId}"`
   - 字段: id, nodeType, resourceType, parentId, deptId, name, userId, lastUpdated
   - `DeptMemberCount` 数据类: deptId + cnt
   - `CrudOperation` 枚举: CREATE, UPDATE, DELETE

2. **CommonNodeTypeConverters.kt** — Room TypeConverter (NodeType ↔ String)

3. **CommonNodeDao.kt** — Room DAO
   - CRUD: insert/insertAll/update/delete/deleteByResourceType/deleteAll
   - Flow 查询: observeAll, observeByResourceType, observeByNodeType, observeByTypeAndNodeType
   - 树形: observeChildren, observeRootDepts, observeMembersByDept, observeSubDepts
   - 统计: countMembersByDept (suspend), observeDeptMemberCounts (Flow)
   - 数据存在性: countByResourceType, getById, getAllByResourceType

4. **CommonNodeRemoteDataSource.kt** — 网络数据源接口
   - `fetchNodes(resourceType)` — 全量拉取
   - `fetchIncremental(resourceType, since)` — 增量拉取（可选）

5. **CommonNodeRepository.kt** — 仓库层 (核心)
   - **本地优先**: observe*() 直接返回 Room Flow，零额外开销
   - **懒加载**: loadIfNeeded() 检查 Room 有数据直接返回，无数据请求网络
   - **强制刷新**: refresh() 从远端全量拉取 → 原子替换 Room → Flow 自动发射
   - **增量刷新**: refreshIncremental() 基于时间戳拉取变更，不支持则回退全量
   - **并发统计**: getDeptMemberCountsConcurrent() 用 coroutineScope + async 并发查询多部门
   - **CRUD 通知**: handleNotification() / handleNotifications() 批量处理
   - **并发安全**: 每个 resourceType 一把 Mutex，刷新和通知互斥
   - observeDeptMemberCounts() 返回 Flow<Map<deptId, count>>

6. **CommonNodeViewModel.kt** — ViewModel 展示用法
   - `_currentResourceType` StateFlow → flatMapLatest → Room Flow → StateFlow
   - nodes / deptCounts 自动跟随 resourceType 切换，无需手动刷新
   - loadData() / refresh() / handleNotification() action

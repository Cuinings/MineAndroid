## CloudContactOrgAggregator 去重修复

**问题**：`flushBatch()` 和 `processUserDomain()` 中对 `nodesOrder` 和 `childrenMap` 使用无条件追加，导致同一 key 重复写入时索引膨胀，`snapshot()`/`searchMember()` 返回重复节点，`memberCount` 偏高。

**修复方案**：改用 `nodes.putIfAbsent(key, node)` 判断首次写入：
- 首次写入 → 正常追加 `nodesOrder` + `childrenMap`
- 重复 key → 仅 `nodes[key] = node` 更新内容，不修改索引

**修改文件**：`board/proxy/src/main/java/com/cn/board/proxy/CloudContactOrgAggregator.kt`
- `flushBatch()` 函数（批量写入去重）
- `processUserDomain()` 内的根节点写入（域根节点去重）
